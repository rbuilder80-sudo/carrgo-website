## 1. Article Metadata

| Field | Value |
|---|---|
| **Title** | (50-60 chars) Include primary keyword + year + angle |
| **Meta Description** | (150-160 chars) Include primary keyword + benefit + CTA |
| **URL Slug** | /resources/KEYWORD-YEAR-MONTH/ |
| **Primary Keyword** | e.g. "china to uk shipping rates" |
| **Secondary Keywords** | 3-5 long-tail variants |
| **Target Audience** | UK importer / procurement manager / small business owner |

| **Publish Date** | YYYY-MM-DD |
| **Last Updated** | YYYY-MM-DD (refresh monthly minimum) |

## 2. AI/GEO Structure Requirements (MANDATORY)

### A. TL;DR Box (Top of Article)
- Positioned immediately after the intro paragraph
- 5 bullet max summarizing key findings
- Include specific numbers, percentages, and actionable takeaways
- Format: highlighted box with clear visual separation

### B. Question-Mirror Headings (H2s)
- Every H2 should mirror a real user query from:
  - Google "People Also Ask"
  - AnswerThePublic / AlsoAsked
  - ChatGPT/Perplexity query logs
- Format: "How to...", "What is...", "Why does...", "Which..."
- Minimum 3 question-based H2s per article

### C. 50-Word Summary Under Every H2
- Immediately after each H2, provide a 40-60 word direct answer
- This is the exact block AI systems extract for snippets and overviews
- Plain language, no fluff, one key fact per sentence

### D. Comparison Tables
- Minimum 1 comparison table per guide article
- AI systems heavily favor tabular data for citation
- Include: metric, value/range, notes/context

### E. FAQ Section (Bottom of Article)
- Minimum 5 questions
- Questions must be extracted from real search data
- Answers: 40-80 words each, direct and factual
- Must have matching FAQPage schema

### F. HowTo Section (For Procedural Content)
- If article includes steps/process, add numbered HowTo steps
- Each step: name + 1-2 sentence description
- Must have matching HowTo schema

### G. Author Bylines (Mandatory)
- Visible author box at top with:
  - Name, job title, company
  - Years of experience
  - LinkedIn link
- Author bio at bottom with credentials and published work
- Must have matching Person schema with @id

---

## 4. Content Quality Rules

- [ ] **No generic AI fluff** — every sentence must contain specific, actionable information
- [ ] **Original data points** — include at least one proprietary statistic, case study figure, or survey result
- [ ] **External citations** — link to authoritative sources (government, industry bodies, carrier data)
- [ ] **Internal links** — minimum 3 links to other Carrgo pages
- [ ] **Freshness signal** — include "Last Updated" date prominently
- [ ] **E-E-A-T signals** — demonstrate first-hand experience, not generic knowledge
- [ ] **Voice search ready** — include conversational Q&A pairs that match spoken queries

---

## 5. SEO Basics

- [ ] Title tag: 50-60 chars, primary keyword first
- [ ] Meta description: 150-160 chars, include CTA
- [ ] Canonical URL: self-referencing, absolute URL
- [ ] Hreflang: en-gb
- [ ] OG tags: complete set
- [ ] Twitter cards: complete set
- [ ] Image alt text: descriptive, include keyword where natural
- [ ] URL: lowercase, hyphenated, no parameters
